# 엑셀(xlsx) 스타일/색상 파싱 라이브러리 옵션

## 현재 사용
- **ExcelJS** + **JSZip** (테마는 xlsx 내부 `theme1.xml` 직접 파싱)
- 무료 라이브러리 중 셀 스타일(fill, font, border)을 가장 잘 노출하는 편입니다.

## 한계 (무료 라이브러리 공통)
- 엑셀은 색을 **테마 인덱스 + tint**로 저장하고, 무료 라이브러리들은 이를 **RGB로 변환해 주지 않음**.
- 그래서 앱에서 테마 XML을 직접 읽어 변환하는 방식을 쓰고 있음.
- 스타일 상속(cellStyleXfs, xfId)은 ExcelJS도 완전히 보존하지 않아, 일부 셀은 색/테두리가 다르게 나올 수 있음.

## 다른 라이브러리

| 라이브러리 | 스타일/색상 | 비고 |
|-----------|-------------|------|
| **SheetJS Pro** (유료) | ✅ 완전 지원 | 테마·조건부서식 등 정확히 해석. 상용 사용 시 고려. |
| **xlsx-js-style** | ⚠️ SheetJS 포크, 스타일 읽기/쓰기 | 테마는 여전히 theme+tint로 옴. 우리가 쓰는 zip 테마 파싱과 조합 가능. |
| **xlsx-with-styles** | ⚠️ 비슷한 포크 | 스타일 객체 구조가 조금 다를 수 있음. |

## 권장
- **무료로 쓸 때**: 현재처럼 ExcelJS + zip으로 theme1.xml 읽는 방식 유지.
- **색이 여전히 어긋날 때**: 배경은 “밝기 기준”으로 어두운 색은 채우기 없음으로 처리하는 휴리스틱 적용됨(`luminance(bg) >= 70`).
- **상용·정확도 최우선**: SheetJS Pro 검토.
